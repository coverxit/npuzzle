N-Puzzle Problem

Introduction

Using the following three kinds of alogrithms to solve the N-Puzzle problem:

  • Uniform Cost Search.
  • A* with the Misplaced Tile heuristic.
  • A* with the Manhattan Distance heuristic.

The main "driver" funtion should match this pseudo code:

function general-search(problem, QUEUEING-FUNCTION)
nodes = MAKE-QUEUE(MAKE-NODE(problem.INITIAL-STATE))
loop do
if EMPTY(nodes) then return "failure"
node = REMOVE-FRONT(nodes)
if problem.GOAL-TEST(node.STATE) succeeds then
return node
nodes = QUEUEING-FUNCTION(nodes, EXPAND(node, problem.OPERATORS))
end

Structure

Since the assignment requires the code should be kept as general as possible, we first introduce classes needed for a (abstract) general searcher:

  • Problem<StateT, ExpandCostT>, The general (abstract) problem class.

    Template Parameters
    StateTThe state type.
    ExpandCostTThe expanding cost type of operations.

    For 8-puzzle, the StateT is std::array and ExpandCostT is int.

  • PriorityQueue<ValueT>, A priority queue based on std::vector, using STL heap to maintain it.

    Template Parameters
    ValueTThe value type to be used in queue.

    Supports changing comparison function dynamically, which is not supported by the STL version.

It actually constructs a min-heap rather than a max-heap that STL version does.

  • GeneralSearcher<StateT, NodeT, ExpandCostT>, The general (abstract) searcher.

    Template Parameters
    StateTThe state type used in a Problem.
    NodeTThe node type used in a PriorityQueue.
    ExpandCostTThe expanding cost type used in a Problem.

    For 8-puzzle, the StateT is std::array, the ExpandCostT is int.

For a specific problem, which is N-Puzzle problem for this project, we defines use a namespace NPuzzle.

Then we set the fundamental arguments:

  • NPuzzle::demonstration, The n-puzzle demonstration, default 8, could be change to 15, etc.
    • NPuzzle::matrixDemonstration, An n-puzzle demonstration would generate a AxA matrix, where A equals to sqrt(demonstration + 1).
  • NPuzzle::moveCost, The cost of every operation (up, down, left, right) on the blank tile.

In addition, we define some structs and typedefs:

  • NPuzzle::NPuzzleState, The state was stored in a std::array.
  • NPuzzle::NPuzzleNode, The node used in search queue.
  • NPuzzle::NPuzzleCostFunction, The cost function type for g(n) and h(n).

Having all these definitions, we can safely define the searcher for N-Puzzle problem:

  • NPuzzle::NPuzzleProblem, The class describes the N-Puzzle problem.
  • NPuzzle::NPuzzleSearcher, The N-Puzzle searcher, with NPuzzle::NPuzzleState as StateT, NPuzzle::NPuzzleNode as NodeT and int as ExpandCostT.

For the three kinds of searching alogrithms, we have the following corresponding functions:

  • NPuzzle::GetUniformHeuristicCost, Uniform Cost Search, h(n) = 0.
  • NPuzzle::GetMisplacedTileCount, Calculate the count of misplaced tiles.
  • NPuzzle::GetManhattanDistance, Calculate Manhattan distance.

Because the assignment requires recording the total amount of nodes expanded and the max length of the search queue, we construct a solver:

  • NPuzzle::NPuzzleSolver, The solver for N-Puzzle problem.

Now we have all the necessary classes, then we can solve the N-Puzzle problem by calling NPuzzle::NPuzzleSolver::solve() with an initialState.
